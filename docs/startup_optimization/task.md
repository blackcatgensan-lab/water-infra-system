# 水インフラ管理システム：起動プロセスの軽量化とオンデマンドロード

## 概要
- **目的**: 23,000件規模のデータを扱う際の初期ロード負荷を軽減し、タイムアウトやフリーズを防ぐ。
- **アプローチ**: `getInitialData` を「共通マスタのみ」に限定し、施設固有データ（設備・ロケーション・点検ログ等）を施設選択後の `getFacilityData` によるオンデマンドロードへ変更する。また、ロケーション情報のハイブリッド取得ロジック（マスタ優先・設備直接入力フォールバック）を確実に実装する。

## 要件
1. **Server-side (Code.gs)**
   - `getInitialData` の軽量化（施設固有データの除外）。
   - `getFacilityData` の完全実装（ハイブリッドロジック含む）。
2. **Client-side (index.html)**
   - 初期ロード時はマスタのみを取得。
   - 施設選択時に `loadFacilityData` を実行し、ローディングUIを表示。
   - 施設未選択時のUI制御（エンプティステートの表示）。
